# Scope Discipline

- Do only the work described in the active Linear issue.
- Do not overbuild beyond the issue acceptance criteria.
- Do not create UI, schema, repository, or feature work that belongs to another issue.
- Do not implement future milestone work unless the user explicitly asks.
- If a better domain model or architecture appears while working, confirm before expanding the issue scope.
- Keep commits aligned with the active Linear issue.
- If the user asks to attach a change to an existing commit, clarify whether they mean a new follow-up commit or `git commit --amend`.
